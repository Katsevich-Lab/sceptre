# Getting help with sceptre

## Questions and usage support

For questions about using `sceptre`, interpreting outputs, or choosing an
analysis workflow, start a [GitHub Q&A discussion](https://github.com/Katsevich-Lab/sceptre/discussions/categories/q-a).
This keeps answers discoverable for future users. If your question genuinely
cannot be made public because it contains
confidential information, you may email Eugene Katsevich at
[ekatsevi@wharton.upenn.edu](mailto:ekatsevi@wharton.upenn.edu). Please use
email only when a public Q&A discussion is not possible; whenever feasible,
remove sensitive details and ask the question publicly instead.

## Bugs

Report reproducible bugs through [GitHub Issues](https://github.com/Katsevich-Lab/sceptre/issues/new).
A useful bug report includes:

- the installed `sceptre` version and installation source;
- the R version and operating system;
- a minimal reproducible example using simulated or public data;
- the exact error or unexpected result; and
- the expected behavior.

## Feature requests

Propose new capabilities in the [Feature requests discussion category](https://github.com/Katsevich-Lab/sceptre/discussions/categories/feature-requests).
Describe the problem or user need, the desired user-facing behavior, relevant
workarounds or alternatives, and how the capability could be validated.
Maintainers may promote a sufficiently scoped proposal to an issue for
implementation tracking.

## Conduct concerns

- For a Code of Conduct concern, follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Do not include private participant data, credentials, protected health
information, or other sensitive material in public support channels.
